# 添加新公司 & 数据更新指南

回答你问的第三个问题：后续怎么增加公司、怎么自动更新数据。

---

## 一、怎么添加新公司？

### 方法 A：手动添加（最简单，适合偶尔加几家）

一共要改 4 个数据文件，都在 `data/` 文件夹里：

#### 1. `data/companies.json` - 公司基础信息

复制一条已有的公司数据，改成新公司的信息：

```json
{
    "id": 21,
    "name": "公司中文名",
    "name_en": "Company English Name",
    "full_name": "公司全称",
    "full_name_en": "Company Full English Name",
    "stock_code_sh": "600000",
    "stock_code_hk": "00000",
    "sector": "创新药（生物药）",
    "sector_en": "Innovative Drug (Biologics)",
    "description": "公司中文介绍",
    "description_en": "Company English description",
    "website": "https://www.company.com",
    "headquarters": "城市名"
}
```

#### 2. `data/processed/financial_metrics.json` - 财务数据

格式如下，按时间填：

```json
"公司中文名": {
    "2025-12-31": {
      "营业收入": 12345678900,
      "归母净利润": 1234567800,
      "研发费用": 1234567800,
      "销售费用": 2345678900
    },
    "2025-09-30": { ... }
}
```

> 💰 **数值单位是"分"还是"元"？** 我们现在用的是**元**。比如营收10亿，就填 `1000000000`。

#### 3. `data/processed/pipeline_data_bilingual.json` - 管线数据

格式：

```json
"公司中文名": {
    "name_en": "Company English Name",
    "drugs": [
      {
        "name": "产品名（商品名）",
        "name_en": "Generic Name (Brand Name)",
        "type": "PD-1单抗",
        "type_en": "PD-1 mAb",
        "stage": "已上市",
        "stage_en": "Marketed",
        "target": "PD-1",
        "target_en": "PD-1",
        "indication": "肺癌、胃癌、肝癌",
        "indication_en": "Lung cancer, gastric cancer, liver cancer",
        "approval_date": "2023-01"
      }
    ]
}
```

#### 4. `data/processed/stock_quotes.json` - 行情数据

```json
"公司中文名": {
    "最新价": 50.25,
    "涨跌幅": 2.35,
    "总市值": 500000000000,
    "市盈率": 25.6,
    "市净率": 3.2
}
```

> 💡 **嫌麻烦？** 少量公司手动加就行。如果以后要加很多家，用脚本自动抓，看下面。

## 二、怎么自动更新数据？

### 方案 1：手动更新（MVP 阶段够用）

**频率：** 每季度财报出来后更新一次，或者每月更新一次行情

**操作：**
1. 打开 `scripts/fetch_financial_light.py`
2. 在你本地电脑（Mac）终端运行：
   ```
   cd 到项目目录
   python3 scripts/fetch_financial_light.py
   ```
3. 脚本会自动从东方财富抓取数据，更新 JSON 文件
4. 然后把更新后的文件上传到 GitHub，网站自动更新

### 方案 2：定时自动更新（进阶）

等你用顺手了，可以配置 GitHub Actions，让它**每天自动跑脚本更新数据**。

**原理：**
- GitHub 每天定时触发 → 运行数据采集脚本 → 更新 JSON 文件 → 网站自动刷新

**设置步骤（简单版）：**
1. 在 GitHub 仓库里建一个 `.github/workflows/update_data.yml` 文件
2. 内容大概长这样：

```yaml
name: Update Data
on:
  schedule:
    - cron: '0 2 * * *'  # 每天北京时间早上10点跑
  workflow_dispatch:
jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install pandas requests
      - run: python scripts/fetch_financial_light.py
      - run: |
          git config user.name = "auto-update"
          git add data/
          git diff --quiet && git diff --staged --quiet || (git commit -m "auto update data" && git push)
```

3. 提交后，每天会自动更新

> ⚠️ **注意**：免费版 GitHub Actions 每月有 2000 分钟额度，每天跑一次完全够用。

### 方案 3：接入专业数据源（付费，更稳定）

如果以后数据量变大、对及时性要求高，可以接入付费数据源：

| 数据源 | 价格（大概） | 覆盖内容 |
|--------|-------------|----------|
| 同花顺 iFinD | 几万/年 | A股全量财务、行情、管线 |
| 恒生聚源 | 几万/年 | A股+港股财务、行业数据 |
| Wind | 几万/年 | 最全的金融数据 |
| Pharmcube 药渡 | 几万/年 | 药品审批、临床试验、管线 |
| 医药魔方 | 几万/年 | 医药行业专属数据 |

**建议：** 先用免费数据源（东方财富、CDE官网、公司公告）跑通流程，等老板认可了再申请付费数据源的预算。

## 三、新闻怎么自动更新？

### 方案：RSS 订阅 + Python 脚本每日抓取

可以抓的免费源：
- **CDE 官网**（药品审批动态）- 有 RSS
- **NMPA 国家药监局** - 官网公告
- **巨潮资讯**（上市公司公告）- 有 API
- **医药魔方、药渡** - 部分免费
- **各公司官网** - 投资者关系栏目

**实现方式：**
1. 写一个 Python 脚本，每天定时抓这些源
2. 提取标题、日期、摘要、关联公司
3. 存成 JSON 文件
4. 自动更新到 GitHub

> 💡 这个功能等 MVP 跑稳了再加就行，不急。新闻是锦上添花，核心还是公司+管线+财务数据。
